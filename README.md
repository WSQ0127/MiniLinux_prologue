# MiniLinux: Prologue

![logo](/img/logo.png)

本项目由 [Sci_qud](https://wsq127.top)/[zMr](https://blog.517group.cn) 开发 qwq。

游戏名暂定为 `MiniLinux: Prologue`。

> 说明：当前仓库仍是开发中的原型；本 README 记录桌游规则设计稿，程序实现可能尚未覆盖全部规则。


## 运行方式

```bash
# 编译（Linux/macOS）
g++ -std=c++17 main.cpp -pthread -o minilinux

# 本机多人轮流游玩
./minilinux

# 局域网开房，默认端口示例为 5555
./minilinux --host 5555

# 其他玩家在同一局域网加入房间
./minilinux --connect <房主局域网IP> 5555
```

联机模式采用房主权威模型：房主程序保存目录树、牌堆、回合状态等完整游戏状态；每位远程玩家通过 TCP 连接接收自己的回合提示并输入操作。

## —— 权限、路径与信息战

---

## 一、游戏简介

在一台共享的 Linux 主机中，数名用户正在争夺最后的生存空间。

你需要：

* 隐藏自己的文件
* 搜索他人的目录
* 获取更高权限
* 利用复杂路径误导敌人
* 在混乱的系统环境中活到最后

当你的用户目录中不再存在任何文件时，你将被系统判定为：

```text
Segmentation Fault
```

并淘汰出局。

最终仍保有文件的最后一名玩家获胜。

---

## 二、游戏目标

成为最后一名：

* 用户目录中仍拥有至少一个文件
* 且未被淘汰

的玩家。

---

## 三、基础概念

### 1. 文件（File）

文件是你的核心资源。

* 文件数量 = 生存能力
* 文件全部丢失即淘汰

文件可被：

* 移动
* 隐藏
* 删除
* 创建

### 2. 目录（Directory）

目录用于：

* 隐藏文件
* 构建复杂路径
* 误导其他玩家

目录本身不能被删除。

### 3. 当前路径

每位玩家拥有一个：

```text
当前路径
```

代表自己当前所在目录。

只有位于当前目录时：

* 才能操作其中内容
* 才能继续进入下一级目录

### 4. 权限

默认情况下：

* 你只能修改自己的目录
* 对其他玩家目录只有只读权限

即：

| 操作 | 默认权限 |
| --- | --- |
| cd | 允许 |
| ls | 允许 |
| rm | 禁止 |
| mv | 禁止 |
| mkdir | 禁止 |
| echo | 禁止 |

若使用：

```text
sudo
```

则下一张卡获得 Root 权限。

## 五、初始设置

### 1. 创建系统结构

游戏开始时：

```text
Home/
 ├── player1/
 ├── player2/
 ├── player3/
 └── ...
```

每位玩家：

* 拥有一个用户目录
* 在根目录放置：

```text
5 个文件
```

文件名可自由命名。

例如：

```text
notes
homework
secret
aaa
test
```

### 2. 初始手牌

每位玩家抽：

```text
5 张手牌
```

### 3. 初始位置

所有玩家初始位于：

```text
/Home/自己的用户目录
```

## 六、回合流程

每位玩家依次进行回合。

### 1. 摸牌阶段

抽：

```text
2 张牌
```

若牌堆为空：

* 洗混弃牌堆
* 重新形成牌堆

### 2. 出牌阶段

你可以按任意顺序使用卡牌。

但存在以下限制：

| 卡牌 | 使用限制 |
| --- | --- |
| cd | 最多 3 次 |
| sudo | 不限 |
| 其余卡 | 各最多 1 次 |

### 3. 弃牌阶段

你的手牌上限为：

```text
max(5, 2f)
```

其中：

* `f` 为你当前拥有的文件数

超出部分需弃置。

---

## 七、卡牌规则

### 1. cd

#### 功能

移动当前路径。

#### 使用方式

每使用一次：

* 只能移动一级目录

例如：

```text
/a/b/c
```

移动到：

```text
/a/b
```

需要：

```text
1 张 cd
```

移动到：

```text
/a
```

需要：

```text
2 张 cd
```

### 2. ls

#### 功能

查看当前目录内容。

#### 可查看

* 子目录
* 文件

#### 默认限制

默认：

* 无法查看隐藏目录

即：

```text
.secret/
```

不会显示。

#### sudo + ls

可查看：

* 所有隐藏内容

等价于：

```text
ls -a
```

### 3. mkdir

#### 功能

创建子目录。

#### 规则

只能在当前目录创建。

目录名自由决定。

例如：

```text
.cache
nothing_here
taxes
tmp
```

### 4. mv

#### 功能

移动文件。

#### 可移动

* 当前目录中的任意文件

#### 可移动到

* 上一级目录
* 任意子目录

#### 重要

只能移动：

```text
当前目录
```

中的文件。

不能隔空移动。

### 5. rm

#### 功能

删除文件。

#### 默认

只能删除：

* 自己当前目录中的文件

#### sudo + rm

可删除：

* 其他玩家文件

#### 限制

一次只能删除：

```text
1 个文件
```

### 6. echo

#### 功能

创建一个文件。

#### 规则

* 在当前目录创建
* 文件名自由命名

例如：

```text
todo
backup
virus
nothing
```

### 7. sudo

#### 功能

使：

```text
下一张卡
```

获得 Root 权限。

#### 规则

* sudo 本身不产生效果
* 必须立刻指定强化目标

例如：

```text
sudo + rm
sudo + mv
sudo + ls
```

### 8. pwd

#### 功能

查看任意玩家当前路径。

#### 例如

```text
/Home/player2/.cache/tmp
```

### 9. chmod

#### 功能

修改一个文件权限。

#### 权限类型

| 权限 | 效果 |
| --- | --- |
| r | 只读 |
| w | 可修改 |
| x | 可执行 |

#### 特殊规则

##### 只读文件

具有：

```text
r
```

的文件：

需要：

```text
sudo + rm
```

才能删除。

##### 可执行文件

具有：

```text
x
```

的文件：

可在自己回合中销毁并发动一次：

```text
额外 ls
```

或：

```text
额外 cd
```

## 八、隐藏机制

### 隐藏目录

以：

```text
.
```

开头的目录：

属于隐藏目录。

例如：

```text
.secret
.cache
.tmp
```

### 默认情况下

普通 `ls` 无法发现隐藏目录。

### sudo + ls

可以发现。

## 九、淘汰规则

若某玩家：

```text
用户目录内不存在任何文件
```

则立即淘汰。

---

## 十、事件牌

每轮结束后：

当前玩家需翻开一张事件牌。

事件立即生效。

### 事件示例

#### Kernel Panic

本轮所有玩家不能使用 `sudo`。

#### Auto Backup

所有玩家复制一个文件。

#### Permission Denied

本轮 `rm` 无效。

#### Disk Failure

所有玩家随机弃置一个文件。

#### System Update

所有玩家摸两张牌。

#### Cache Overflow

所有隐藏目录暴露直到下一轮开始。

## 十一、胜利条件

当场上仅剩：

```text
1 名玩家
```

未被淘汰时：

该玩家获胜。

---

## 十二、推荐玩法（非常重要）

### 1. 不公开目录结构

玩家：

* 不公开自己的目录树
* 不公开文件位置

仅公开：

* 文件数量
* 当前手牌数量

### 2. 路径需要真实记录

建议：

* 使用纸笔
* 或小型目录板

记录自己的文件系统。

### 3. 鼓励欺骗

例如：

```text
/homework/final/real/not_here/tmp
```

可能根本没有文件。

### 4. 文件名可以误导

例如：

```text
virus
do_not_open
empty
```

可能只是普通文件。

---

## 十三、推荐扩展规则（可选）

### 1. 符号链接（ln）

创建假路径。

### 2. 病毒文件

删除时触发反击。

### 3. Root 用户

某玩家暂时获得全局权限。

### 4. Shell Script

组合多张卡一起发动。

---

## 十四、设计理念

MiniLinux 并不是：

> “谁删得更快”

而是：

> “谁更会隐藏、侦查与误导”。

你并非在管理文件。

你是在：

```text
管理信息
```
